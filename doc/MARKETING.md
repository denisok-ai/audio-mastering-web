# Magic Master (magicmaster.pro) — маркетинговый пакет

Живой документ для рекламных кампаний, брифов подрядчикам и внутренней команды.  
**Канонические цифры по тарифам и лимитам** — из бэкенда (`payments.py`, `config.py`, `deps.py`, `mastering.py`), а не с лендинга.

---

## 1. Краткое описание продукта

### Одна строка
**Magic Master — профессиональный мастеринг аудио в браузере за минуту: загрузили трек, выбрали стиль, скачали результат.**

### Питч из трёх предложений
1. Сервис автоматически приводит громкость к стандартам стриминга (LUFS), ограничивает True Peak, выравнивает спектр студийным EQ и многополосной динамикой — без установки DAW.
2. Поддерживаются WAV, MP3, FLAC; есть жанровые пресеты, PRO-модули (деноайзер, де-эссер, transient designer, parallel comp, dynamic EQ, reference match), AI-консультант и Telegram-бот.
3. Можно начать **без регистрации**; платные тарифы Pro и Studio дают токены на мастеринг, расширенные лимиты файлов и очередь.

### Отличия от типичных конкурентов (LANDR, eMastered, BandLab Mastering и др.)
- Русскоязычный интерфейс и поддержка рынка СНГ из коробки.
- Набор PRO-модулей в одном продукте (reference match, dynamic EQ, spectral denoiser и др.) — в рекламе подчёркивать «глубину» цепочки, а не только «громче/тише».
- Telegram-бот: мастеринг и анализ **с телефона** без открытия сайта.
- Позиционирование под **AI-музыку** (Suno, Udio): быстрый путь от сырого рендера до релизного LUFS.
- Открытый веб + PWA (установка как приложение).

## 3. УТП (уникальные торговые предложения)

Используйте в креативах и на лендинге (формулировки можно сокращать):

- **Без регистрации** для первого знакомства (гостевой режим на сайте).
- **8 жанровых стилей** мастеринга + платформенные ориентиры LUFS (стриминг, Apple, YouTube, клуб, broadcast).
- **PRO-модули**: спектральный деноайзер, де-эссер, transient designer, parallel compression, dynamic EQ (8 полос), reference match, streaming loudness preview (в интерфейсе).
- **Цепочка DSP**: многополосная компрессия, лимитер/максимайзер, нормализация LUFS, дизеринг (TPDF и др.), защита от межвыборочных пиков.
- **Telegram-бот**: мастеринг, анализ, пресеты, AI-чат, баланс, история — после привязки аккаунта.
- **AI**: рекомендация пресета, отчёт, чат-консультант (при включённой функции в админке); лимиты по тарифам.
- **Форматы**: вход WAV/MP3/FLAC; выход WAV (+ MP3/FLAC/OPUS/AAC на платных тарифах).
- **Изоляция вокала (Demucs)** — опционально, если включено на сервере.
- **Пакетная обработка** — для Studio (лимиты по API/фронту).
- **REST API** — тариф Studio (для интеграций и продакшн-пайплайнов).
- **PWA** — установка с главного экрана телефона.
- **Оплата в РФ** — ЮKassa, цены в рублях.

## 4. Матрица возможностей по тарифам (канон для маркетинга)

| Параметр | Free / гость | Pro | Studio |
|----------|----------------|-----|--------|
| **Регистрация** | Не обязательна для гостя | Да | Да |
| **Мастеринг без оплаты (гость)** | До **1 мастеринга в неделю на IP** (сброс по логике бэкенда) | — | — |
| **Токены** | Не начисляются автоматически при регистрации; платные планы дают пакет токенов | Подписка: **50 токенов/мес** или **600/год** (см. оплату) | **100/мес** или **1200/год** |
| **Докупка токенов** | — | 100 ₽ = 1, 500 ₽ = 5, 1000 ₽ = 10 | То же |
| **Дневной лимит мастерингов (после списания токена)** | — | **10/сутки** (UTC) | **30/сутки** (UTC) |
| **Макс. размер файла (типовой лимит тарифа)** | **100 МБ** | **300 МБ** | **800 МБ** |
| **AI-запросы/сутки (конфиг)** | **5** | **50** | **Без лимита** (−1 в конфиге) |
| **Очередь** | Обычная | Приоритет (семафоры бэкенда) | Приоритет |
| **API, batch** | Нет | По продуктовой матрице фронта | Да (Studio) |

**Важно:** на лендинге и в `pricing.html` местами указаны «3 мастеринга в день» и другие лимиты — это **не совпадает** с текущим гостевым лимитом (1/неделя по IP). Для рекламы до выравнивания сайта используйте формулировку: **«попробуйте бесплатно, без карты»** без конкретной цифры или синхронизируйте тексты с разработкой.
