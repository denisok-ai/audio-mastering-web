# Плагины и расширения

Минимальная архитектура расширяемости: пресеты сообщества и задел под будущие расширения.

## 1. Пресеты сообщества (расширяемые)

- **Встроенные:** `backend/app/presets_community.json` — читается при каждом запросе к `GET /api/presets/community`.
- **Дополнительные:** переменная окружения `MAGIC_MASTER_COMMUNITY_PRESETS_EXTRA` — путь к одному JSON-файлу или к каталогу с файлами `.json`.

Поведение:

- Сначала загружаются пресеты из `presets_community.json`.
- Если задан `COMMUNITY_PRESETS_EXTRA`:
  - **Файл** — загружается один файл (должен быть массив объектов пресетов).
  - **Каталог** — загружаются все `.json` из каталога (каждый файл — массив пресетов).
- При слиянии дубликаты по полю `id` отбрасываются (приоритет у первого встреченного).

Формат пресета см. в `backend/app/PRESETS_COMMUNITY_README.md`.

Пример (доп. файл рядом с проектом):

```bash
export MAGIC_MASTER_COMMUNITY_PRESETS_EXTRA="/opt/masterflow/presets_extra.json"
```

Или каталог с несколькими файлами:

```bash
export MAGIC_MASTER_COMMUNITY_PRESETS_EXTRA="/opt/masterflow/community_presets"
```

## 2. API расширений

- **`GET /api/extensions`** — минимальный статус расширений (без авторизации):
  - `community_presets_extra_configured` — задана ли переменная `COMMUNITY_PRESETS_EXTRA`;
  - `community_presets_extra_loaded` — успешно ли загружен файл/каталог (существует и пригоден).

Используется для отладки и мониторинга (например, в админке или статус-странице).

## 3. Будущие расширения (задел)

- **Процессоры:** возможность подключать дополнительные аудио-процессоры (например, плагины в отдельном каталоге с контрактом «функция получает буфер и опции, возвращает буфер») — в текущей версии не реализовано, архитектура пайплайна в `pipeline.py` допускает добавление шагов.
- **Пресеты по URL:** загрузка списка пресетов по HTTP — при необходимости можно добавить отдельный параметр конфигурации и подгрузку по расписанию или при старте.

Текущий фокус — расширяемые пресеты сообщества через файлы/каталог и минимальный API статуса.
