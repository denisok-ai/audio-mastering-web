# Чек-лист ручной проверки (п.9 плана)

Используйте при ручной проверке админки и критичных сценариев основного приложения.

## Основное приложение (главная страница)

- [ ] **Загрузка файла** — выбор WAV/MP3; отображается волновая форма, метаданные (длительность, sample rate).
- [ ] **Измерить громкость** — кнопка активна при загруженном файле; после нажатия появляется индикатор «Измерение…»; при успехе отображаются LUFS, пик; при ошибке — тост с текстом ошибки (не пустая «Внутренняя ошибка» без деталей).
- [ ] **Спектр / Векторскоп** — после замера отображаются данные или сообщение «Нет данных. Загрузите файл и нажмите Измерить громкость».
- [ ] **AI отчёт** — при загруженном файле кнопка активна; по нажатию запрос к API; при успехе показывается блок с отчётом; при ошибке — тост с `detail`.
- [ ] **Рекомендовать пресет (AI)** — при загруженном файле кнопка активна; по нажатию приходит ответ; пресет подставляется или показывается сообщение.
- [ ] **Пресеты (кнопки)** — выбор пресета из списка применяет настройки цепочки; кнопка «Запустить мастеринг» доступна.
- [ ] **Запустить мастеринг** — отправка формы; появляется индикатор обработки; при успехе — `job_id`, опрос статуса, по завершении — возможность скачать результат; при ошибке (в т.ч. 429 лимит) — тост с текстом из ответа (`detail`), не «500» без пояснений.
- [ ] **Сравнение до/после** — после мастеринга доступны A/B и LUFS; кнопка **Upscale 96k** скачивает апсемпленный файл или показывает тост с ошибкой.
- [ ] **Авто-мастеринг** — кнопка «Мастер» (авто) при загруженном файле; при исчерпании лимита (429) — тост с сообщением о лимите Free-тарифа, а не 500.

## Админ-панель (/admin)

- [ ] **Вход** — email и пароль из `.env` (MAGIC_MASTER_ADMIN_EMAIL / MAGIC_MASTER_ADMIN_PASSWORD); при неверных данных — сообщение в блоке ошибки.
- [ ] **Выйти** — кнопка «Выйти» в шапке; по нажатию токен сбрасывается, показывается форма входа.
- [ ] **Dashboard** — загрузка статистики (пользователи, Pro, транзакции и т.д.) без ошибок в консоли и тостах.
- [ ] **Пользователи** — список, пагинация, поиск; открытие карточки пользователя; смена тарифа; при ошибке API — тост с текстом.
- [ ] **Транзакции** — список транзакций отображается.
- [ ] **Новости** — список, создание/редактирование поста; сохранение без 500.
- [ ] **Настройки** — сохранение настроек (если есть форма); при ошибке — тост с `detail`.
- [ ] **Отчётность** — выбор отчёта, таблица с данными; сортировка по колонкам (если реализована).
- [ ] **LLM (Подключение / Промпты)** — отображение и сохранение без 500; при ошибке — тост с текстом.

## После проверки

Отметьте пройденные пункты и зафиксируйте найденные баги (страница, действие, ожидание, факт). При 500/429 проверьте логи сервера (`journalctl -u magic-master -n 100`) и ответ API (вкладка Network в DevTools).
