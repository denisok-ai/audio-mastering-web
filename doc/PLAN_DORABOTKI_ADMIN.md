# План доработок админ-панели

> **Источник:** [AUDIT_ADMIN_PANEL.md](AUDIT_ADMIN_PANEL.md) (аудит по замечаниям: выход из учётной записи, API ключи, настройки).  
> **Обновлено:** 2026-03

---

## Цель

Исправить нерабочие или неочевидные функции админ-панели: выход из учётной записи, сохранение и использование API ключей, обратная связь по настройкам.

---

## Очередь 1 — Выход из учётной записи

| # | Задача | Описание | Статус |
|---|--------|----------|--------|
| 1.1 | **Корректный выход и отображение формы входа** | При нажатии «Выйти»: очищать токен и `localStorage.admin_token`; скрывать основной layout (`.layout` — `display: none` или `visibility: hidden`); показывать overlay с формой входа (`display: flex`). После успешного входа — снова показывать layout. | ✅ |
| 1.2 | **Видимость кнопки «Выйти»** | Убедиться, что кнопка «Выйти» в topbar всегда видна (не обрезается на малых экранах, не перекрывается). При необходимости вынести в отдельную область или добавить иконку. | ✅ |

---

## Очередь 2 — API ключи (LLM)

| # | Задача | Описание | Статус |
|---|--------|----------|--------|
| 2.1 | **Обратная связь при сохранении ключей** | После PATCH настроек LLM: при успехе показывать тост «Ключи сохранены» и перезагружать блок «Подключение (LLM)», чтобы отображался актуальный статус (какой бэкенд, задан ли ключ). При ошибке — показывать текст ответа сервера (detail), а не только «Ошибка». | ✅ |
| 2.2 | **Отображение статуса подключения** | В блоке «Подключение (LLM)» явно показывать: «OpenAI: ключ задан / не задан», «DeepSeek: ключ задан / не задан», «Активный бэкенд: …». Использовать данные из GET /api/admin/settings (llm.openai_key_masked и т.д.). | ✅ |
| 2.3 | **Проверка подключения (опционально)** | Кнопка «Проверить подключение»: лёгкий запрос к бэкенду (например, эндпоинт /api/admin/llm/test или вызов существующего AI-метода без расхода лимита); отображать результат «Подключение успешно» / «Ошибка: …». | ✅ |

## Очередь 3 — Настройки (применение и понятность)

| # | Задача | Описание | Статус |
|---|--------|----------|--------|
| 3.1 | **Подсказки в разделе «Настройки»** | Добавить краткий текст: «Настройки из вкладок Общие, SMTP, YooKassa, Telegram, LLM сохраняются в БД и применяются без перезапуска. Исключение: CORS, JWT secret и часть параметров берутся из .env при старте сервера.» | ✅ |
| 3.2 | **Обратная связь после «Сохранить»** | После успешного сохранения любой вкладки настроек показывать тост с указанием секции, например: «Сохранено: Общие», «Сохранено: SMTP». При ошибке — выводить detail из ответа. | ✅ |
| 3.3 | **Предупреждение о перезапуске (опционально)** | Для настроек, которые не применяются без перезапуска (если такие есть в UI), выводить пометку «Вступает в силу после перезапуска сервера». | ✅ |
